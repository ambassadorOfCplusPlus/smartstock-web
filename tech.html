Технические детали

Для тех, кому интересно, как SmartStock устроен внутри. На остальных страницах — по-человечески; здесь — по-инженерному.

Стек и архитектура

⚙️

Нативное приложение

C++23 и Qt6 — это не браузер и не Electron. Отсюда быстрый запуск, отзывчивый интерфейс и скромный аппетит к памяти.

🗄️

Локальная база

Встроенная база SQLite в вашем профиле пользователя. Никакого сервера баз данных ставить не нужно.

🧩

Один файл — один склад

Десктоп рассчитан на один компьютер и один склад. Для сети точек — серверная версия с синхронизацией.

Данные и безопасность

🔒 Шифрование базы

База шифруется целиком (SQLCipher, AES-256). Файл лежит у вас на диске — не в чужом облаке.

💾 Резервные копии

Автоматический онлайн-бэкап со сжатием; хранятся 30 последних. Восстановление — в пару кликов.

🧾 Журнал действий

Все значимые операции пишутся в журнал аудита — видно, что и когда менялось.

📁 Данные отдельно от программы

База, копии и настройки хранятся в профиле пользователя — переустановка и обновление их не трогают.

Интеграции

🛒

Маркетплейсы

Wildberries и Ozon по схеме FBS: выгрузка остатков и цен, импорт заказов с автосписанием по партиям. Сопоставление товара — по штрих-коду; для цен WB номер карточки (nmID) подтягивается автоматически из карточек продавца.

🧾

Касса 54-ФЗ

Фискализация через официальный «АТОЛ Драйвер ККТ v10». При недоступной кассе чек уходит в очередь допробития и пробивается позже.

🤖

Локальный ИИ

Языковая модель (llama.cpp, ~2,5 ГБ) работает прямо на вашем ПК, без API-ключей и передачи данных. CPU-движок подбирается под процессор автоматически (новые — AVX2, старые — AVX/SSE).

Навигация по складу — как считается путь без GPS

Модуль SmartStock Навигация (открытый, warehouse-navigator) ведёт к ячейке внутри помещения, где спутниковый GPS не работает. Это гибрид точной привязки и счисления пути.

МеханизмКак работает
QR-якоряМетки с известными координатами на стеллажах и входах. Скан любой метки сбрасывает накопленную ошибку позиции в ноль (абсолютная привязка).
Счисление пути (PDR)Между якорями положение считается по датчикам телефона: направление — фьюжн компаса и гироскопа, перемещение — детектор шагов по акселерометру.
Длина шага (Вайнберг)Оценивается по амплитуде вертикальных колебаний, а не берётся фиксированной — точность выше у людей с разным ростом и походкой.
Отсев магнитных аномалийМеталл и стеллажи искажают компас; выбросы магнитного поля отбрасываются, чтобы не уводить курс.
Map-matchingТрек «притягивается» к проходам плана склада — маршрут не проходит сквозь стены и стеллажи.
СтекЧистое TS-ядро навигации без рантайм-зависимостей (покрыто тестами) · лёгкий LAN-сервер на Node · мобильное приложение на Expo React Native · ПК-клиент на Qt6/C++.

Шифрование переписки (мессенджер)

Открытый модуль мессенджера со сквозным шифрованием: сервер видит только шифротекст и публичные ключи.

🔐 Сквозное шифрование (E2E)

Сообщение шифруется на устройстве отправителя публичным ключом получателя (X25519 + XSalsa20-Poly1305, формат NaCl crypto_box). Расшифровать может только адресат.

📦 Вложения

Файл шифруется случайным ключом на устройстве (secretbox); на сервер уходит непрозрачный блоб, ключ к файлу сервер не получает никогда.

🕵️ Sealed-sender

Сообщение уходит без подписи отправителя — автора определяет только получатель после расшифровки. Метаданные «кто кому» скрыты от сервера.

🛡️ Защита метаданных

Выравнивание длины сообщений, скрытые реакции и вложения, отпечатки ключей для защиты от подмены собеседника (MITM).

🧰 Библиотеки

Проверенные примитивы NaCl/libsodium. Веб-клиент — tweetnacl, десктоп (Qt/C++) — libsodium; единый формат шифрования совместим между ними.

🖥️ Клиенты

Веб — один статичный HTML-файл (открывается в браузере, без установки). Десктоп — нативное приложение на C++/Qt.

Серверная версия — как устроена синхронизация

🖧

Узел синхронизации

Node.js + PostgreSQL (данные), Redis (кеш и очереди), MinIO (файлы, вложения, установщики). Разворачивается в Docker; ставится на один компьютер или VPS.

📦

Персональные сборки (sidecar)

Установщик сотрудника — неизменённый exe + конфиг рядом в ZIP (схема sidecar), а не «склейка хвостом». Роль и права прописываются за миллисекунды без пересборки программы.

🤖

Локальный ИИ

Модель phi-4-mini работает на сервере, без облака и платы за запросы. По желанию подключается внешний OpenAI-совместимый провайдер (например, DeepSeek).

🌐

Доступ снаружи

Cloudflare-туннель даёт доступ к серверу из любой сети без выделенного («белого») IP. Адреса подключений можно не логировать.

🔄

Модель синхронизации

Offline-first: журнал событий, идемпотентность по UUID, vector clocks; конфликты — физический (недостача → инвентаризация) и смысловой (голосование).

📜

Открытый код

Ядро синхронизации, навигатор, мессенджер и движок списания — отдельные MIT-модули. См. страницу про открытый код.

Системные требования

  • Windows 10 или 11 (64-бит)
  • 4 ГБ ОЗУ (для ИИ-модели желательно 8 ГБ+)
  • ~200 МБ на диске (+2,5 ГБ для ИИ-модели, по желанию)
  • Принтер — любой, видимый Windows; сканер штрих-кодов — USB «в разрыв клавиатуры»
  • Касса 54-ФЗ: ККТ АТОЛ + «Драйвер ККТ v10» (по желанию)

Ограничения — честно

  • Десктоп: один компьютер, один склад (сеть точек — сервер)
  • Маркировка — локальный учёт кодов, без обмена с ГИС МТ
  • ЕГАИС (алкоголь) не поддерживается
  • Установщик без платного сертификата — при первом запуске Windows SmartScreen покажет «неизвестный издатель» (это нормально, в инструкции описано)

Качество: проект покрыт 204 автоматическими тестами; исходный код частично открыт — см. страницу про открытый код.
