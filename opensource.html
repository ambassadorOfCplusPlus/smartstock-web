Модули

Пять самостоятельных частей, выделенных из проекта SmartStock. Каждая автономна, со своей лицензией MIT, README и тестами.

💬 messenger

Самостоятельный E2E-мессенджер: сервер, веб-клиент и нативный Qt-клиент. Группы, файлы, сквозное шифрование — приватный ключ не покидает устройство, веб- и Qt-клиенты совместимы по формату шифрования.

Node / Fastify / Prisma · vanilla JS (tweetnacl) · Qt6 / libsodium

🧮 writeoff-engine

Движок партионного учёта: списание по партиям FIFO / LIFO / средневзвешенная, резервы, возвраты. Не привязан к базе данных — хранилище подключается через интерфейс BatchStore.

TypeScript (+ оригинал на C++ в cpp-reference/)

✅ 14 / 14 тестов

🔄 sync-engine

Offline-first синхронизация (референс-дизайн + ядро): журнал событий, идемпотентность по UUID, vector clocks, два типа конфликтов — физический (недостача → инвентаризация) и смысловой (голосование).

TypeScript (ядро) + DESIGN.md

✅ 40 / 40 тестов

🛒 marketplace-adapters

Адаптеры маркетплейсов (FBS): Wildberries / Ozon / Яндекс.Маркет / Сбер — выгрузка остатков и цен, импорт заказов. HTTP-клиент инъектируется, ключи продавца не хранятся в библиотеке.

TypeScript, ноль рантайм-зависимостей

📦 installer-glue

Склейка установщика: дописывает в конец готового base.exe хвост с манифестом и файлами — персональный установщик за миллисекунды, без перекомпиляции. Библиотека + CLI.

Node.js (только встроенные модули)

Что внутри интересного

Кросс-платформенная E2E-крипта

Один формат шифрования (crypto_box / secretbox) работает и в вебе (tweetnacl), и в нативном Qt (libsodium). Групповой «пер-получательский конверт», гибрид для файлов.

⚖️

Два типа конфликтов

Сначала спроси, что конфликтует — факт или мнение. Двойное списание остатка не разрулить голосованием (это недостача), а две разные цены — наоборот, решаются голосованием. Разные стратегии для разных случаев.

⚡

Склейка вместо пересборки

Лишние байты после полезной нагрузки PE/ELF загрузчик игнорирует — exe остаётся валидным. Персональная сборка под пользователя за миллисекунды, без тулчейна на сервере. CRC32 zlib-совместим: один формат, две реализации.

Нужна готовая программа, а не код?

Открытые модули — для разработчиков. Если нужен готовый складской учёт «поставил и работай» — это десктоп, сервер для сети точек или облако по подписке.
