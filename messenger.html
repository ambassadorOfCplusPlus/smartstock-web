Smartstack Messenger

Мессенджер со сквозным шифрованием, который вы хостите сами — на своём сервере или компьютере. Сервер хранит только шифротекст: переписку и файлы не может прочитать никто, включая владельца сервера. Работает отдельно от SmartStock — это самостоятельный продукт.

⭐ Исходники на GitHub Запросить развёртывание

Открытый код под лицензией MIT — шифрование можно проверить, а не принять на веру. Развёртывание в Docker одной командой.

🔐 Сквозное шифрование (E2E) 🏠 Self-hosted — на вашем железе 📜 Открытый код (MIT) ✅ 17 автотестов

Что это

Полностью автономный мессенджер: сервер и два клиента. Не зависит от складской программы — можно поставить только его.

🖥️

Свой сервер

Node.js + PostgreSQL, разворачивается в Docker. Реестр аккаунтов, диалоги, хранение зашифрованных сообщений и вложений. Видит только шифротекст и публичные ключи.

🌐

Веб-клиент

Один статичный HTML-файл, шифрование на tweetnacl. Откройте в браузере и укажите адрес своего сервера — ничего ставить не нужно.

💻

Десктоп-клиент

Нативное приложение на C++/Qt (шифрование на libsodium). Один формат шифрования с вебом — клиенты полностью совместимы.

Возможности

💬

Личные и групповые чаты

1-на-1 и группы. В группах сообщение шифруется отдельно под каждого участника — общего «группового ключа», который можно перехватить, нет.

📎

Зашифрованные файлы

Файл шифруется случайным ключом на устройстве, на сервер уходит непрозрачный блоб. Ключ к файлу сервер не видит никогда.

🔑

Ключ не покидает устройство

Пара ключей X25519 у каждого пользователя. Публичный — на сервере, приватный хранится только у вас (в браузере или в файле ключа Qt).

🕵️

Сервер не знает автора

Sealed-sender: сообщение отправляется без «подписи» отправителя — кто написал, определяет только получатель. Плюс выравнивание длины, реакции и файлы тоже скрыты, без отметок «прочитано», IP не логируется.

🔢

Отпечатки ключей

Сверьте отпечаток ключа собеседника по другому каналу — и подмену ключа сервером (MITM) будет видно.

🔎

Поиск по короткому ключу

Контакты находятся по короткому идентификатору или имени. Регистрация — по ключу, который выдаёт администратор инстанса.

👥

Лимит аккаунтов на инстанс

Администратор задаёт, сколько аккаунтов на сервере, и видит оценку нагрузки. Удобно для команды или клуба по интересам.

🐳

Развёртывание в Docker

Одна команда — поднимаются PostgreSQL и сервер, миграции применяются сами. Подойдёт VPS или компьютер в офисе.

Как работает шифрование — простыми словами

У каждого пользователя — пара ключей. Публичный ключ уходит на сервер, приватный остаётся на устройстве и никогда его не покидает.
Сообщение шифруется на вашем устройстве публичным ключом получателя (X25519 + XSalsa20-Poly1305). На сервер уходит только шифротекст.
Получатель расшифровывает сообщение у себя своим приватным ключом. Между ними сервер выступает лишь «почтовым ящиком» для запечатанных конвертов.
Файлы шифруются отдельным случайным ключом, который передаётся внутри того же конверта, — сервер хранит лишь зашифрованный блоб.

Современные, проверенные примитивы NaCl/libsodium. Веб (tweetnacl) и десктоп (libsodium) используют один формат — сообщение, отправленное из браузера, читается в Qt-клиенте, и наоборот.

Честно о приватности

Сильная сторона — конфиденциальность содержимого: переписку и файлы нельзя прочитать на сервере. Но честно обозначим и границы — настоящая приватность строится на том, что проверяемо, а не на лозунгах.

✅ Чего сервер НЕ видит

  • Текст сообщений — только шифротекст
  • Содержимое файлов — только зашифрованный блоб
  • Приватные ключи и ключи файлов — они остаются на устройствах

ⓘ Что сервер всё-таки видит

  • Публичные ключи и кто с кем в диалоге (метаданные)
  • Время сообщений и размеры шифротекста/вложений
  • IP-адреса подключений — это не анонимная сеть (не Tor)

То есть это сквозное шифрование содержимого, а не анонимность. Чем это сильно: вы хостите сервер сами, данные не уходят третьим лицам, а шифрование можно проверить по открытому коду. Внешнего аудита безопасности пока не проводилось — код открыт именно для того, чтобы его можно было изучить и проверить.

Тем, кому важна приватность переписки
